## A strategic portal for monitoring displacement sites

Developed for the UN Refugee Agency (UNHCR), the <a href="https://im.unhcr.org/apps/sitemapping/#/" target="_blank">Site Mapping</a> application serves as the central reference for visualizing and monitoring refugee and internally displaced persons (IDP) sites globally. The primary objective is to provide a global and updated view of the situation on the ground to support intervention coordination.

The major challenge was to offer standardized mapping visualization despite the heterogeneity of field surveys and sites. alongside the application's development, a site mapping campaign via the <a href="https://missingmaps.org" target="_blank">Missing Maps</a> project was launched to use OpenStreetMap data when data in the UNHCR database was missing.

## Technical implementation

- **Technical stack**: Web interface developed with Vue, Mapbox-GL-JS, and Turf.js, with an ArcGIS Server backend for UNHCR data.
- **Indicators**: The application calculates KPIs on health and security themes. These indicators are calculated on the client side using Turf via web-workers to avoid blocking navigation during significant calculations.
- **OSM**: To detect sites containing relevant OSM data, <a href="https://2021.stateofthemap.org/sessions/XLDXW9/" target="_blank">a correspondence work between the NGO's data model and OSM tags</a> was performed. Once this tag list was established, a PostGIS database containing all OSM data with these tags was implemented using Imposm. This database is updated in real-time (fetching diffs every minute). Automated ArcPy scripts then calculate the indicators on the OSM data and inject them into the ArcGIS Server database.
